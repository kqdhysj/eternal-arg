# 救援调查报告高拟真公文设计

## 目标

将 `rescue-report.html` 从影视道具式“红色机构页”调整为乍看符合中国党政机关公文习惯的上行文页面。保留“深空航行事故联合调查组”这一虚构机构、原剧情正文、表格信息、勘查附注、缓存入口和终端返回路径，不使用真实机关名称、国徽或现存机关印章。

## 方案选择

比较三种方向：

1. **党政机关上行文拟真（采用）**：红色机关标志、红色分隔线、左发文字号与右签发人、标准标题与正文层级、成文日期、虚构圆章和版记。识别度最高，也与“关于……情况的报告”文种匹配。
2. **事故调查简报**：信息密度高、表格更多、标题较克制，但更像内部工作材料，不足以回应“高仿红头文件”。
3. **国务院公报式附件**：正式但缺少红头与签发结构，更接近公开发布稿，不符合绝密现场报告的叙事身份。

## 页面结构

- 页面外层使用冷灰背景，中央呈现接近 A4 比例的白色纸张；桌面端保留纸张边界，移动端改为满宽连续文档。
- 缓存徽章置于纸张上方，终端操作提示和返回入口置于纸张下方，明确属于 ARG 缓存界面而非公文正文。
- 公文首页从上到下依次为：份号与密级、红色机关标志、红色分隔线、发文字号与签发人、标题、正文、署名与成文日期、虚构印章、现场勘查附注、版记。
- 删除拼音副标题、巨幅“绝密”框、红底白字横幅和满屏高对比水印，避免影视道具感。

## 视觉规范

- 机关标志为“深空航行事故联合调查组文件”，使用小标宋风格字体回退栈、正红 `#d40000`，不添加国徽。
- 发文字号采用 `深空联调〔97〕9号`；报告作为上行文，发文字号在左，签发人在右。
- 标题使用小标宋风格；正文使用仿宋风格；一级标题黑体，二级强调楷体。正文保持两字符首行缩进和接近公文排版的行距。
- 红色分隔线使用单条主线并带中央五角星作为虚构机关版头识别符号；不复刻任何真实机关标志。
- 表格取消灰色填充，以细黑线、仿宋正文和明确列宽呈现。
- 署名、成文日期和圆章位于正文右下。圆章只写虚构机构名称，使用 CSS/SVG 绘制，不使用真实印章图样。
- 版记使用上下分隔线，包含抄送机关、印发机关与印发日期；ARG 报告编号可保留为归档字段，但与版记分区。

## 内容边界

- 不改写四个正文部分、现场发现表格和勘查附注的叙事内容。
- 保留现有页面链接与文件名，不新增 JavaScript 行为。
- 可将现有说明文字重新归位或增加“附件/附注/归档编号”等结构标签，但不改变线索含义。
- 机构、公文编号、时间均保持世界观内虚构，不伪装成真实政府文件。

## 响应式与可访问性

- 桌面端以接近 A4 的内容宽度和页边距展示；窄屏按比例缩小页边距、字号和版头字距，禁止横向滚动。
- 链接保留清晰的悬停与键盘焦点状态。
- 打印时隐藏缓存徽章和终端返回区，移除纸张阴影与外部背景。
- 装饰性印章和分隔符使用合适的无障碍隐藏属性，不干扰正文阅读顺序。

## 验收标准

- 首屏不再出现拼音副标题、红色实心横幅、巨幅绝密框或密集水印。
- 版头、发文字号、签发人、标题、正文、落款、成文日期、印章与版记顺序完整。
- 原剧情正文、表格信息、附注、缓存说明和终端链接均存在，线索含义不变。
- 桌面端与 390px 窄屏均无横向溢出；移动端版头不换成难以辨认的碎行。
- 页面可直接打开，无控制台错误；返回 `index.html` 的链接可用。
