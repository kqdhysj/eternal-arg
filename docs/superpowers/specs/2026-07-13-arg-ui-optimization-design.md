# 永恒号 ARG UI 优化设计

## 目标

在不改变 ARG 文本、线索、密码、路径和交互逻辑的前提下，优化项目内全部 HTML 页面。终端相关页面以林汐蓝 `#88ccff`、深空黑 `#0a0e17` 和硬边界面为核心；外部网页继续保留新闻、博客、文档、商业站点等各自身份，只统一可读性、缓存来源提示和交互反馈。

## 实施原则

- 直接修改现有 HTML 与内嵌 CSS，不新增共享样式依赖。
- 不修改 `app.js`。主页来源识别通过 `index.html` 中独立的展示后处理脚本完成。
- 不修改故事内容、交互入口、文件路径、密码提示、日志文本或页面链接关系。
- 桌面端优先，保持固定宽度或 `max-width`，不新增响应式重排。
- 终端界面使用直角或不超过 `2px` 的圆角，不使用阴影、毛玻璃、渐变按钮或复杂入场动画。
- 外部页面保留原有视觉主题，但正文不得使用纯白大段文字；正文最小字号为 `14px`，行距不低于 `2.0`。

## 页面设计

### 终端主页 `index.html`

- 登录区改为战舰舱门认证面板：增加结构标识、状态线、输入字段标签和硬边分隔。
- 输入框使用低强度林汐蓝边缘高亮；聚焦时增强边框，不使用阴影。
- 登录失败利用 `.login-msg:not(:empty)` 触发短促红色闪烁，并尊重减少动态效果设置。
- 终端输出行距提升至 `2.0`，同时保留现有颜色语义。
- 时钟改成带 `SYS.TIME` 标识的独立状态块；正常时间使用林汐蓝，异常值保留 glitch 效果并提高可见度。
- 在 `app.js` 之后增加展示脚本，使用 `MutationObserver` 监听 `#terminal` 的 `childList` 新增行，不采用定时轮询：
  - 当前路径位于 `/mnt/0x2B`，或输出命中 `0x2B` 连接特征时，添加 `source-node-2b`。
  - 当前路径位于 `/mnt/7F-A3`，或输出命中 `7F-A3` 连接特征时，添加 `source-node-7f`。
  - 输出命中缓存列表、缓存打开和 `/cache` 结果特征时，添加 `source-cache`。
  - 对应左边框分别为警告橙、林汐蓝、暗淡灰。
- 后处理脚本只添加展示类，不修改命令、状态、内容或事件处理。

### 陈远博客 `blog.html`

- 保留泛黄纸张与工程日志身份。
- 强化现有左侧时间轴和节点层级，不重建文章结构。
- 移除 `backdrop-filter`、卡片阴影和超过 `2px` 的卡片圆角。
- 文件路径、节点编号、命令和设备编号使用等宽技术样式；技术引用块与普通回忆引用块产生清晰区分。
- 保留归档开关、涂黑文字和导航交互。

### Tick 日志 `tick-log.html`

- 页面背景、导航、页头和日志面板统一为深空终端色系。
- 日志条目增加纵向间距和左侧级别提示，INFO/WARN/CRIT/DATA 保持各自颜色。
- 筛选项改为硬边按钮；活动项同时使用边框、文字和背景反馈，不能只依赖颜色。
- 搜索框增加命令提示符视觉和明确聚焦态，保留现有即时过滤逻辑。
- 保留导出失败、打印失败以及日志筛选交互。

### 发射新闻 `launch-news.html`

- 保留新闻门户布局、下拉导航、热搜、评论区、推荐内容和现有可交互元素。
- 正文首段增加首字下沉；引文改为更接近新闻专题的高亮引用。
- 去除循环背景、滑入和淡入类复杂动画；保留即时 hover 与展开反馈。
- 新闻白色区域改为轻微冷调纸色，避免纯白大面积背景。
- 评论区的删除态、操作入口和折叠功能保持原逻辑。

### 沉浸式文档页面

范围：`archive.html`、`node-7f.html`、`passenger-51.html`、`relay-7f-a3.html`、`rescue-report.html`。

- 保持档案、私人日记、未知节点、工程拆解报告和官方救援报告的不同身份。
- 正文字号不低于 `14px`，行距不低于 `2.0`。
- 纯白或纯黑底色改为带冷暖倾向的纸色或深空黑。
- 底部返回终端入口改为带边线、明确文字和 hover/focus 状态的行动区。
- 保留涂黑内容、印章、表格、ASCII 图和所有线索文本。

### 其他外部页面

范围：`about-xinhua.html`、`quantumcloud.html`、`deepcomm.html`、`kepler-realty.html`。

- 保留机构介绍、意识保险、深空通讯和星际地产的原有主题；其中 `about-xinhua.html` 继续保持冷淡、克制的官方机构视觉，不引入终端配色。
- 校准文本对比度、正文行距、链接 hover/focus 和硬边按钮。
- 去除不符合全局禁忌的阴影、过大圆角和非必要复杂动画。
- 不把这些页面强行改造成船舰终端。

## 缓存徽章与全局一致性

- `/cache` 可直接打开的八个页面统一增加窄条 `.cache-badge`：`blog.html`、`archive.html`、`launch-news.html`、`node-7f.html`、`tick-log.html`、`relay-7f-a3.html`、`passenger-51.html`、`rescue-report.html`。
- 徽章固定采用深空黑背景、暗淡灰说明、林汐蓝链接和直角分隔，统一放在页面最顶部、现有导航栏上方，不覆盖页面自身导航。
- 全部页面标题逐一检查并保持唯一、可辨识。
- 全部链接增加一致的 `:hover` 和 `:focus-visible` 反馈；颜色随页面主题调整，但终端链接优先使用林汐蓝。

## 数据流与错误处理

- `app.js` 继续负责登录、命令执行、终端输出和页面打开。
- `index.html` 后处理脚本仅通过 `MutationObserver({ childList: true })` 观察已经插入 DOM 的输出节点，并根据当前提示符与文本添加来源类。
- 脚本在目标节点缺失、非元素节点或未知输出时直接跳过，不抛出错误，不阻断终端。
- 搜索、筛选、评论、导航等现有内联事件保持不变；样式修改不得改变 `display`、数据计数或点击目标。

## 验收标准

- 所有 13 个 HTML 页面可独立打开，无 HTML 解析错误和控制台异常。
- `app.js` 的文件内容及 Git 哈希保持不变。
- 登录成功、登录失败、访客模式和命令输入正常。
- 0x2B、7F-A3 与缓存输出获得正确来源色条；普通终端输出不被误标。
- Tick 的 INFO/WARN/CRIT/DATA 筛选和搜索可以组合使用，活动状态清晰。
- 新闻导航、评论折叠、热搜、弹层和相关交互仍可用。
- 博客归档开关与涂黑文字揭示正常。
- 所有缓存页面的徽章和返回入口可见，键盘焦点清晰。
- 文档正文满足 `14px` 与 `2.0` 的最低可读性要求。
- 页面视觉检查确认无纯白终端背景、无黑客绿、无渐变按钮、无毛玻璃、无过度阴影和复杂入场动画。
